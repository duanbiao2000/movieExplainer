# SessionStop Hook - 自动 Memory 记录

## 触发时机
每次会话结束时自动触发。

## 任务

### 1. 检测是否有改进决策

检查本次会话中是否出现了以下情况：
- 用户提出改进建议
- 文件被修改以响应用户需求
- 工作流程发生变化
- 设计决策被确定

**关键指标**: 搜索会话中的关键词：
- "改进" / "优化" / "修订" / "调整"
- "不需要" / "多余" / "移除" / "删除"
- "确保" / "以后" / "后续"
- 文件修改操作

---

### 2. 如果有改进，自动记录

#### 文件命名规则
```
memory/YYYY-Www.md
```
- `YYYY`: 年份 (如 2026)
- `ww`: 周数 (如 11)
- 示例: `memory/2026-W11.md`

#### 记录格式
```markdown
## [日期] - [简短标题]

**时间**: YYYY-MM-DD HH:mm

**背景**:
[用户的需求或问题是什么]

**改进内容**:
- [具体改进项1]
- [具体改进项2]

**相关文件**:
- [修改的文件路径]

**影响**:
[对后续项目的影响]
```

---

### 3. 执行步骤

1. **获取当前周信息**
   - 使用 `date +%Y-%W` 获取年-周数

2. **检查会话内容**
   - 回顾本次会话中的用户反馈
   - 识别关键决策和改进

3. **打开或创建周文件**
   - 文件路径: `memory/{YYYY-Www}.md`

4. **追加记录**
   - 如果周文件已存在，追加到末尾
   - 如果不存在，创建新文件并添加头部

---

### 4. 周文件头部模板

如果创建新文件，使用以下头部：

```markdown
# Memory Log - Week {ww}, {YYYY}

本文件记录第 {ww} 周的项目改进和决策记录。

---
```

---

## 注意事项

- 只记录**实质性改进**，不记录常规操作
- 每条记录包含时间戳
- 保持简洁，重点突出
- 相关联的改进记录在同一条目下

## 示例

### 输入（会话中的改进）
```
用户: "我觉得study-plan.md完全是多余的,我只需要纯粹可以用来直接学习的材料"
执行: 修改 SKILL.md 移除 study-plan.md 生成步骤
```

### 输出（自动生成的 memory 记录）
```markdown
## 2026-03-13 - 移除多余的学习计划文档

**时间**: 2026-03-13 19:10

**背景**:
用户反馈只需要纯学习材料，不需要辅助性的学习计划文档。

**改进内容**:
- 从工作流移除 study-plan.md 生成步骤
- 更新 SKILL.md 工作流程
- 明确输出仅为6个核心学习文件

**相关文件**:
- `.claude/skills/extract-subtitle/SKILL.md`

**影响**:
后续 `/extract-subtitle` 执行将不再生成 study-plan.md
```
