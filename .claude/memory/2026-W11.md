### 改进 7: 添加 Anki 卡片音标支持

**背景**:
用户发现 Anki 卡片中缺少音标，影响发音学习效果。
> "我发现anki卡片中没有音标. 为保证测验效果. 建议将音标放在背面卡片"

**设计决策**:
1. **保留两字段模型** (Front/Back) - 保持 Anki 模板稳定
2. **音标在内容生成阶段加入** - 保持 Python 转换脚本简练
3. **仅单词类型卡片需要音标** - 其他类型（短语、句子）不需要

**音标规则**:
| 卡片类型 | 是否添加音标 |
|---------|-------------|
| `vocabulary-*` | ✅ 是 |
| `topic-*` | ✅ 是 |
| `verb-phrase-*` | ❌ 否 |
| `expression-*` | ❌ 否 |
| `key-point-*` | ❌ 否 |
| `collocation-*` | ❌ 否 |

**音标格式**: `<b>音标</b>: /音标内容/<br>` （置于 Back 字段最前）

**解决方案**:
1. 更新 `content-extractor/agent.md` - 添加 CSV 音标格式规范
2. 更新 `SKILL.md` - 在"步骤 6"中明确包含音标的 CSV 格式

**相关文件**:
- `.claude/agents/content-extractor/agent.md`
- `.claude/skills/extract-subtitle/SKILL.md`

**影响**:
- 词汇和话题类卡片将包含音标，帮助发音学习
- 短语和句子类卡片保持简洁，无音标干扰
